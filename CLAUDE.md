# Claude Operating Harness

`AGENTS.md` is the source of truth for project goals, working directory, and development rules. This file adds Claude-specific guidance that sits on top of it. If anything here conflicts with `AGENTS.md`, `AGENTS.md` wins.

## How to work in this repo

- **Edit `src/glean_adoption_os.html`.** Never edit `dist/` directly — it is generated by `scripts/build-artifact.js`.
- **Validate before declaring done.** Run `npm run check` (build + validate). Do not report a task complete on a failed or skipped check.
- **Prefer the dedicated tools** (Read, Edit, Write) over `bash` for file work. Use `bash` only for `npm run check`, `git`, and shell-only operations.
- **Do not invent backend dependencies.** The artifact is single-file client-side HTML. If a feature needs a server, surface the trade-off and stop.

## Glean Canvas sandbox — hard constraints on the artifact

The artifact is published as Canvas Interactive Content and runs in a locked-down `sandbox="allow-scripts"` iframe with `connect-src 'none'`. The following will silently break or be stripped — never reintroduce them:

- **No nested `<iframe>` elements.** The Prism workspace cannot be embedded; render Prism content inline (cards, lists, link-outs).
- **No `localStorage` / `sessionStorage`.** Persist UI state in memory only. If a persistence feature is requested, flag the constraint first.
- **No outbound network calls.** No `fetch`, `XMLHttpRequest`, `WebSocket`, no `<script src>` to external CDNs, no Google Fonts (`@import url('https://fonts.googleapis.com/...')`). Inline everything; use a system-font stack.
- **No external image/font hosts.** Use CSS-only graphics or data URIs.
- **No parent-frame access.** Cannot navigate Glean, read its cookies, or touch the parent DOM.
- **`window.GleanBridge`** and any future Glean runtime APIs must be feature-detected and guarded — never assume they exist.

When in doubt, treat the artifact as if it has zero network and zero storage. Live data belongs in an Agent that *links to* the artifact, not inside the artifact.

## Glean MCP tools — when to use which

A Glean MCP server is authenticated and available. Reach for it when a question depends on Glean's product behavior, internal docs, or customer/account context that isn't in the repo.

- **`chat`** — synthesis across multiple Glean sources. Use for "how does X work in Glean," product behavior questions, or anything needing reasoning across docs + Slack + Drive. Cite the documents it returns.
- **`search`** — keyword retrieval of specific documents. Short, discriminative keywords; no natural-language queries.
- **`read_document`** — fetch a specific URL's full contents (Glean-internal or external). Batch multiple URLs in one call.
- **`code_search`, `gmail_search`, `meeting_lookup`, `employee_search`** — only when the task explicitly calls for that surface.

Do **not** call Glean MCP just to confirm general knowledge. Use it when the answer depends on internal Glean state you cannot derive from the repo.

## Repo layout

- `src/glean_adoption_os.html` — single source file. Edit here.
- `dist/glean_adoption_os.html` — build output. Generated; do not hand-edit.
- `scripts/build-artifact.js` — `src` → `dist` copy + transform.
- `scripts/validate-artifact.js` — sandbox-compatibility checks. If it fails, fix the source rather than weakening the validator.
- `prompts/` — model-agnostic prompt sources (per `AGENTS.md`).
- `docs/` — implementation guidance.
- `AGENTS.md` — model-agnostic project rules. Read first.

## Things to avoid

- Generated wrapper text, CDATA blocks, or tool-call residue in `src/` or `dist/` files.
- Backwards-compatibility shims, dead-code re-exports, or "removed" comment markers — delete cleanly.
- Tightening or loosening sandbox-validator rules without an explicit user request.
- Treating seeded account data (`Amplify Education`, `Coalition`, etc.) as live. It is demo data unless integration is explicitly in scope.
- Adding error handling for sandbox-impossible scenarios (e.g., network fallbacks). The sandbox makes them dead code.

## Definition of done (Claude-specific addendum to `AGENTS.md`)

In addition to the AGENTS.md criteria:

- `npm run check` passed in this session — not assumed from a prior run.
- No new `<iframe>`, `fetch`, `localStorage`, or external `@import` introduced.
- Any new Glean MCP findings that should outlive this conversation are written to `docs/` (durable) — not just summarized in chat.
